[h2] 🚀 How to Contribute
[h3] Option 1: Suggest a Real Quote (Easiest!)
[li] Browse the Events Timeline
[li] Click on an event you know about
[li] Look for the placeholder/summary badges in the Reactions section
[li] Click "💡 Found a Real Quote? Suggest it here"
[li] Fill out the form with the quote, author, source link, and date
[li] Submit - it creates a GitHub issue for review
[h3] ✅ Good sources to check:
[li] LessWrong - AI safety researcher discussions
[li] EA Forum - Effective altruism community
[li] Scott Alexander's blog - ACX coverage of AI safety
[li] Twitter/X - Real-time reactions from researchers
[li] News articles - Fortune, MIT Tech Review, TechCrunch, etc.
[h3] Option 2: Fix Event Metadata
[p] If you notice errors in categories, tags, rarities, or game impacts:
[li] Click the event detail page
[li] Scroll to "🏷️ Event Metadata" section
[li] Click "→ Suggest different [category/rarity/tags]"
[li] Opens GitHub issue - explain what should change and why
[h3] Option 3: Bulk Review (Advanced)
[p] If you're familiar with the AI safety landscape:
[li] Use the Events Browser table view
[li] Filter by category (e.g., "funding_catastrophe")
[li] Select multiple events with checkboxes
[li] Click "Bulk Suggest Metadata"
[li] Review events systematically by topic area
[h2] ✅ Quote Quality Guidelines
[h3] ✅ Good Quotes
[p] Safety Researcher Reactions:
[li] Direct quotes from known AI safety researchers
[li] Comments on LessWrong, EA Forum with author attribution
[li] Blog posts analyzing the event's impact
[li] Conference presentations or papers citing the event
[p] Media Reactions:
[li] Headlines from major publications (Fortune, MIT Tech Review)
[li] Excerpts from news articles covering the event
[li] Public statements from organizations
[li] Social media reactions from verified accounts
[h3] ❌ Bad Quotes
[p] Don't submit:
[li] Paraphrased or summarized content (unless it's your own summary)
[li] Anonymous sources or unverified accounts
[li] Private messages or non-public communications
[li] Quotes taken out of context
[li] Dead links or paywalled content
[h3] 📋 Required Information
[p] When suggesting a quote, you must provide:
[li] ✅ Quote text - The exact words (no paraphrasing)
[li] ✅ Author - Who said/wrote it
[li] ✅ Source URL - Direct link to the quote
[li] ✅ Date - When it was published
[li] ✅ Platform - LessWrong, Twitter, blog, news, etc.
[p] Optional but helpful:
[li] Context (e.g., "Comment on LessWrong thread about FTX collapse")
[li] Your name (for credit if quote is accepted)

[h2] ❓ Questions
[h3] Why are there so many placeholders?
[p] The 949 alignment research events were imported from a database that didn't include reaction quotes. We're now working with the community to source real reactions from the literature.
[h3] Can I suggest multiple quotes for one event?
[p] Yes! Major events often have multiple notable reactions. Submit them separately and we'll track them all. In the future, we may implement a quote carousel for events with multiple high-quality reactions.
[h3] What if I don't have a GitHub account?
[p] You can email quotes to team@pdoom1.com with the subject "Event Quote Suggestion". Include all the required information (quote, author, source, date).
[h3] Will my contributions appear in the game?
[p] Yes! Real quotes will be used in:
[li] Event card displays when events are drawn
[li] Historical timeline view in the game
[li] Character dialogue and commentary
[li] Event notifications and logs
